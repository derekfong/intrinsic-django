Django - Written Comparison

Django is an open-source web framework built using Python. Django was built primarily with the intent to be used to develop web applications that require a database back-end. It is ideal to use Django to create web applications that interact with sensitive user information. The main aspects of Django that developers will find beneficial are as follows: highly modular, developer-friendly, development efficient, encouragement of reusability, portability, and importance of security.

Modularization is a technique used to separate a large component into smaller parts/modules that are well-defined. Django demonstrates modularity in that it strictly follows the Model-View-Controller (MVC) software architecture. However, Django does make a slight modification to the architecture. In Django, the traditional View is called a Template and the traditional Controller is called a View. The purpose of using a templating system is to modularize the tasks of code development, design and content.

Web frameworks should be developer-friendly, which means that they should be easy for developers to learn, code, and manage. Django is developer-friendly in that it allows a developer to begin immediately after Django is setup. It comes preloaded with an integrated lightweight developer web server and the ability to use SQLite as the database backend. This means if a developer wants to start developing right away, they don’t have to consume their time setting up a web server (e.g. Apache) or a database server (e.g. MySQL). They can setup Django on their local machines and jump right into coding. This setup is also extremely useful for testing, as the developer doesn’t have to use their production web server to test web application changes or updates. They can conveniently use the built in web server and SQLite to test on their isolated local machine. One of the features of Django that makes administering a web application seemless is the built-in administrative interface. When enabled, the administrative interface provides the administrator with a simple, yet powerful web interface for managing the web application.

The efficiency of a web framework is a very important when choosing which framework to use when developing a web application. Fortunately, Django is quite development efficient. In particular, Django significantly reduces the lines of code written. The major benefit of this is that it reduces the chances of making human errors (e.g. typos) when coding, mainly because there is a smaller subset of code a developer would have to look over when reviewing code.

Aspects of Django

  1. Modularity (showing code)
    • Strictly follows the MVC (Models-Views-Controllers) software architecture.
    • Django slightly modifies the MVC structure by using Templates for the Views.
    • Templating system
  2. Developer-friendly (admin interface, ETC)
    • Comes preloaded with a lightweight web server.
    • Comes preloaded with SQLite.
    • Allows a developer to start developing right away on their local machine with minimal setup.
    • Built-in administrative interface for seamless management of web application with minimal effort.
  3. Development Efficiency (lines of code for admin interface, easier way to do things)
    • Significantly reduces the amount of code written.
    • Less chance of making syntactical errors.
  4. Reusability (explain the loose coupling)
    • Created applications can be utilized in other applications if needed.

  5. Flexibility(URL Design)

    • Pretty, customized URLs
  6. Security (built in)
    • SQL Injection - the Django database API automatically protects against SQL injection by escaping certain SQL parameters.

Rails:

Ben Askins and Alan Green presented “A Rails / Django Comparison” at the Open Source Developer’s Conference in 2006, which was later published as a peer reviewed paper. Both developers implemented the same specifications for a web application, one in Ruby on Rails and one in Django; the results were compared quantitatively and qualitatively.

Both applications were compared based on lines of code and time spent developing each portion of the application. Overall, Django was only minimally ahead of Ruby on Rails in terms of lines of code (726 vs. 814); however, Django was well ahead of Ruby on Rails in lines of code for Ruby and Python implementation (320 vs. 517). Django lagged significantly behind Ruby on Rails in templating system implementation (406 vs. 297). In terms of time spent on implementation, Django again bested Ruby on Rails; implementation in Django was nearly half of the time spent on Ruby on Rails. The largest difference between the two implementations was in the administration pages. This is an area that Django really exceeds compared to other web frameworks. In Django, only a small number of lines of code are required to build and customize an administration interface
